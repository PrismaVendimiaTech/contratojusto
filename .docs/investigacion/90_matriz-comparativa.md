# Matriz Comparativa de Tracks

Actualizado: 2026-03-25

## Ranking rapido para alguien que hoy no sabe blockchain

1. `Stellar Ideathon`
2. `Hedera Ideathon`
3. `Beexo Connect`
4. `BNB Chain`
5. `Hedera Hackathon`
6. `Stellar Hackathon`
7. `Rootstock`

## Tabla comparativa

| Track | Accesibilidad | Que exige de verdad | Mejor para | Riesgo principal | Recomendacion |
| --- | --- | --- | --- | --- | --- |
| Stellar Ideathon | Muy alta | problema validado + deck + pitch + por que Stellar | remesas, becas, pagos | hacer una idea demasiado abstracta | excelente puerta de entrada |
| Hedera Ideathon | Alta | caso real + negocio + justificar Hedera | pagos agenticos, ops, ONG | sonar enterprise sin caso concreto | muy buena opcion |
| Beexo Connect | Alta | integracion `xo-connect` y UX clara | cobros, pagos QR, UX LATAM | quedarse solo en "wallet connect" | ideal para frontend fuerte |
| BNB Chain | Media | demo funcional + contrato + 2 tx onchain | bots, mini apps, consumer AI | no cumplir la prueba tecnica | buena si puedes ejecutar un contrato simple |
| Hedera Hackathon | Media | pagos via agentes + demo tecnica | asistentes de pago | demasiado tooling | buena si acotas fuerte |
| Stellar Hackathon | Media | app funcional con stack Stellar | rails de pago y activos | meterte demasiado rapido en Soroban | viable si simplificas |
| Rootstock | Baja-media | DeFi sobre Bitcoin + contrato o integracion real | ahorro, colateral, finanzas | producto demasiado complejo | usar solo si Bitcoin agrega valor |

## Lectura por criterio

### Mejor track para ganar aprendiendo

- `Stellar Ideathon`

### Mejor track para una demo de impacto social seria

- `Stellar`
- `Hedera`
- `Beexo`

### Mejor track para un producto tipo bot o asistente

- `BNB Chain`
- `Hedera`

### Mejor track para UX local LATAM

- `Beexo Connect`

### Mejor track si quieres hablar de Bitcoin y dinero duro

- `Rootstock`

## Recomendacion de estrategia, no solo de track

### Estrategia A: menor riesgo

- Diseñar para `Stellar Ideathon`
- Si el MVP tecnico aparece, extender a `Stellar Hackathon`

### Estrategia B: impacto + AI

- Diseñar para `Hedera Ideathon`
- Bajar a `Hedera Hackathon` solo si logras un flujo agentico muy acotado

### Estrategia C: UX primero

- Diseñar un caso social local y competir por `Beexo Connect`
- Si el modelo lo amerita, sumar Rootstock minimo

## Lo que no recomiendo

- Empezar por `Rootstock DeFi` sin una hipotesis financiera muy clara.
- Elegir `BNB Chain` si todavia no puedes asegurar dos transacciones reales.
- Elegir `Stellar Hackathon` pensando solo en "aprender Rust sobre la marcha".

## Mejor decision hoy

Si hubiera que tomar una decision de una sola linea hoy:

`Diseñar una idea social que cierre como Stellar Ideathon, pueda crecer a Hedera/Beexo si hace falta, y solo luego decidir si baja a MVP tecnico.`
