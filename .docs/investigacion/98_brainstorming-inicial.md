# Brainstorming Inicial

Actualizado: 2026-03-25

## North Star

Queremos un producto que:

- sea util para la sociedad
- no dependa de saber mucho de blockchain hoy
- tenga historia de usuario clara
- permita una demo realista antes del cierre
- pueda defender por que usa blockchain y no solo una base de datos comun

## Oportunidades donde blockchain si agrega valor

### 1. Cuando hay multiples actores que no confian entre si

Ejemplo:

- universidad
- ONG
- beneficiario
- comercio

### 2. Cuando el estado del dinero importa

Ejemplo:

- fue asignado
- fue reclamado
- fue gastado
- fue liquidado

### 3. Cuando una wallet o rail interoperable simplifica pagos

Ejemplo:

- remesas
- micropagos
- vouchers
- retiro o cobro por app

### 4. Cuando la IA reduce friccion

Ejemplo:

- onboarding
- elegibilidad
- explicacion del estado del tramite
- soporte conversacional

## Oportunidades que NO conviene priorizar en esta hackathon

- tokenizacion compleja sin usuarios reales
- DeFi sofisticada si todavia no dominas colateral y riesgo
- trazabilidad por trazabilidad sin problema de negocio fuerte
- idea "marketplace web3" generica

## Direcciones de producto con mas potencial

### Direccion A: Ayuda y becas

Pregunta central:

`Como hacemos que una ayuda llegue mejor, se entienda mejor y se audite mejor?`

Ventajas:

- valor social clarisimo
- jurado lo entiende rapido
- demo facil

Desventajas:

- hay que cuidar privacidad y no sonar invasivo

### Direccion B: Remesas y pagos familiares

Pregunta central:

`Como bajamos incertidumbre y friccion en pagos entre personas?`

Ventajas:

- Stellar fit natural
- problema enorme en LATAM

Desventajas:

- si prometes demasiado, tocas compliance y operaciones reales muy pesadas

### Direccion C: Cobro para trabajadores informales

Pregunta central:

`Como hacemos que cobrar, guardar y entender la plata sea mas simple para quien hoy esta fuera del sistema financiero formal?`

Ventajas:

- gran impacto social
- Beexo fit fuerte
- UX demoable

Desventajas:

- requiere una propuesta mejor que "otro link de pago"

## Idea recomendada

## Ayuda Clara

### Problema

Hoy muchas becas, viaticos y ayudas de emergencia son:

- opacas para quien las recibe
- dificiles de auditar para quien las entrega
- lentas de gestionar
- poco claras en estado, monto y siguiente paso

### Usuario principal

- estudiante o beneficiario

### Usuario operador

- universidad
- ONG
- cooperativa
- programa social pequeño

### Propuesta de valor

`Ayuda Clara` permite asignar, seguir y cobrar una ayuda con trazabilidad simple y una experiencia entendible, apoyada por un asistente que explica el estado del beneficio en lenguaje humano.

## MVP 48h

### Flujo minimo

1. Una institucion crea un programa de ayuda.
2. Da de alta beneficiarios o solicitudes.
3. Se aprueba una ayuda.
4. El sistema genera un estado verificable.
5. El beneficiario ve:
   - monto
   - estado
   - pasos siguientes
6. Se demuestra un claim, pago o comprobante.

### Que mostrar en demo

- panel muy simple de admin
- vista de beneficiario
- asistente o FAQ AI
- una accion blockchain real o una explicacion muy concreta de como ocurriria segun el track

## Como se adapta por track

### Stellar version

- vouchers o pagos
- historia fuerte de rails
- posibilidad de `Ideathon` si la parte tecnica no llega

### Hedera version

- agente que asiste o dispara pagos
- narrativa muy alineada con el track

### BNB Chain version

- mini app o bot de ayuda
- contrato simple de asignacion y claim
- foco consumer AI

### Beexo version

- UX local de cobro o uso de beneficio
- onboarding amigable

### Rootstock version

- solo si quieres agregar ahorro/estabilidad financiera con Bitcoin

## Nombre provisorio del concepto

- `Ayuda Clara`
- alternativas:
  - `Beca Clara`
  - `Fondo Claro`
  - `Trama Social`

## Tesis de pitch en una frase

`Ayuda Clara vuelve entendibles y trazables las becas y ayudas de bajo monto para estudiantes y organizaciones comunitarias, combinando rails digitales y soporte conversacional para reducir friccion, errores y opacidad.`

## Lo que haria inmediatamente despues de esta investigacion

1. Elegir `Ayuda Clara` como idea base.
2. Decidir track primario:
   - `Stellar` si quieres menor riesgo
   - `Hedera` si quieres enfatizar agentes de IA
   - `Beexo` si quieres destacar UX local
3. Armar en una sola hoja:
   - problema
   - usuario
   - demo
   - por que esa blockchain
4. Recien despues definir si vale la pena escribir codigo o cerrar primero una submission tipo ideathon.
